## 3. Existing Katas That Should Remain Vanilla Vim

These should not become LazyVim-first exercises:

- `058`-`065`, `078`, `090`, `091`, `097`: search, regex, `gn`, `cgn`, and substitution grammar are core Vim skills. Add optional LazyVim notes only after the stock workflow.
- `067`, `068`, `074`-`077`, `081`, `082`, `085`, `086`, `088`, `092`-`096`, `111`-`114`: these teach fundamental commands, motions, Ex commands, insert-mode edits, line starts/ends, counts, formatting, sorting, undo, and command-line history.
- `071` and `106`: shell filtering is portable Vim behavior; LazyVim does not replace `:'<,'>!` or `!!`.
- `073`: should remain explicitly built-in `gd`; LSP definition should be covered separately or as a note.
- `084`, `100`, `101`: quickfix/location lists should stay stock because Trouble, pickers, Grug-far, diagnostics, and test tools often build on those lists.
- `089`: diff mode basics should remain built-in; Gitsigns belongs in a separate hunk workflow.
- `087`: built-in completion should remain separate from Blink/nvim-cmp so users understand fallback behavior.
