# LazyVim Compatibility Audit: Katas 001-057

## 1. Batch Scope and Notable Existing Coverage

Scope inspected: `001_*.md` through `057_*.md` only, compared against `lazyvim-kata-audit/01_lazyvim_workflow_digest.md`.

This batch already covers the strongest portable foundation for LazyVim users: Vim grammar, operators, motions, counts, dot repeat, search, text objects, Visual/block mode, Ex ranges, buffers, splits, tabs, marks, jump/change lists, registers, macros, shell filters, `:argdo`, and Vim regex modes. Most exercises correctly use built-in Vim/Neovim behavior and should stay centered on that.

## 4. Gaps Not Covered by This Batch

- LazyVim discovery flow: `which-key`, `<leader>` groups, readiness checks.
- Snacks/picker workflows: find files, recent files, project grep, explorer, scratch.
- Commenting: `gc`, `gcc`, `gco`, `gcO`.
- LSP: `gd`, `gr`, `gI`, `gy`, `K`, `gK`, rename, code actions, organize imports.
- Diagnostics and lists: `[d`/`]d`, severity jumps, Trouble views, quickfix/location fallback.
- Formatting/linting: `<leader>cf`, autoformat toggles, Conform/nvim-lint behavior.
- Git: gitsigns hunks, `ih` hunk text object, preview/stage/reset hunk, LazyGit.
- Completion/snippets: Blink/nvim-cmp-agnostic accept, expand, field-jump flow.
- Testing/debugging: Neotest and DAP extras.
- Treesitter-specific selections, folds, and syntax-aware navigation.

## 5. Proposed Next LazyVim Katas to Add

1. LazyVim discovery and readiness: use `<leader>`, `which-key`, and plugin checks before plugin-dependent drills.
2. Flash navigation: `s`, `S`, operator-pending `r`, and fallback to `/`, `f`, or built-in motions.
3. LazyVim file/buffer navigation: picker, recent files, buffer picker, buffer cycling, buffer delete.
4. `mini.ai` code text objects: change function args, call contents, blocks, classes.
5. Comment workflow: line, operator, above/below comment insertion with `gc` family.
6. LSP edit loop: definition, references, hover, rename, code action, organize imports.
7. Diagnostics loop: create an error, jump diagnostics, open Trouble, fix and verify.
8. Formatting/linting loop: format buffer, format injected language, toggle autoformat.
9. Git hunk workflow: edit, jump hunks, preview, stage/reset, use `ih`.
10. Completion/snippet workflow with engine-neutral tasks and Blink/cmp notes.
